# 汇编
“times 510-($-$$) db 0”
$：代表当前指令的地址。
$$：代表程序的起始地址。
$-$$：计算出从开始到当前位置一共占用了多少字节
这是一个编译器指令，意思是“重复执行”。它会将剩余的空间全部填充为0，直到达到第510字节

“dw”
定义一个字（Define Word），占用 2 字节

汇编代码中，可以在代码的任何地方定义标签和数据（部分全局规则还是习惯放在开头）
1.段地址[org 0x7c00]必须放在开头。org告诉编译器从这一行开始，所有地址计算都要加上0x7c00，如果放在中间，则前半部分代码偏移是0,后半则是0x7c00
2.标签Label的位置不受限制。因为汇编器是多遍扫描：第一遍扫描整个文件记住所有标签，第二遍填充真正的地址
3.写MBR等底层代码时，应“代码在上，数据在下”

在汇编中，%include是一个预处理指令。它的作用是在编译器(nasm)开始编译你的主文件之前，把另一个文件的内容原封不动的复制粘贴到当前位置。用于模块化开发，避免一个代码文件过长降低可读性
1.要特别注意%include放置的位置，这决定了代码在内存里的物理顺序
